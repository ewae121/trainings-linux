Linux Documentation

Linux is extremly well documented generally. This documentation is written by contributors. As the code is Open source it is easy to check if the documentation is correct. For most of Posix commands, the manual pages are precises.

The documentation is available with the man command. "Man" stand for "Manual".

Linux Manpages

A man page (short for manual page) is a form of software documentation usually found on a Unix or Unix-like operating system. Topics covered include computer programs (including library and system calls), formal standards and conventions, and even abstract concepts. A user may invoke a man page by issuing the man command.

You can see a Man page using the man command as following: man [options] [[section] page ...]

Manpages are located in the folder: /usr/share/man in ubuntu.

Whatis command usage example

See the available passwd commands 
          $ whatis passwd
          passwd (5)           - fichier des mots de passe
          passwd (1)           - Modifier le mot de passe d'un utilisateur
          passwd (1ssl)        - OpenSSL application commands
See the available pass* commands 
          $ whatis -w pass*
          passwd (1)           - Modifier le mot de passe d'un utilisateur
          passwd (5)           - fichier des mots de passe
          passphrase-encoding (7ssl) - How diverse parts of OpenSSL treat pass phrases character encoding
          passwd (1ssl)        - OpenSSL application commands
          passwd2des (3)       - RFS password encryption

Get man chapter information

See the available pass* commands 
          $ man passwd | grep -A 2 'SYNOPSIS'
          SYNOPSIS
              passwd [options] [LOGIN]

Navigate in man pages

"q"to quit the manual
"g"Go to the top of the manual
"G"Go to the end of the manual
"/"Regex to find then navigate with "n" to next element and "N" to the previous element.

Man page sections

SectionDescriptionexample
1General commandsman ls
2System callsman 2 fork
3functions, covering in particular the C standard libraryman 3 string
4Special files (usually devices, those found in /dev) and driversman 4 tty
5File formats and conventionsman 5 passwd
6Games and screensaversman 6 kpat
7Miscellaneousman 7 inode
8System administration commands and daemonsman 8 ld.so

POSIX system calls are often present in both sections 2 and 3, where section 2 contains the system call's documentation from the local operating system and section 3 contains the system call's documentation from the POSIX Programmer's Manual.

Unix System V uses a similar numbering scheme, except in a different order(see wikipedia for details).
